# BigQuery Hashtag Reference Guide

Always include `#BigQuery` in every tweet thread. Choose 1-2 additional tags from below based on the update category.

## Category → Hashtag Map

| Update Type      | Primary Tag      | Secondary Tag (if applicable)   |
|-----------------|------------------|---------------------------------|
| Feature          | #GoogleCloud     | #DataEngineering                |
| Announcement     | #GoogleCloud     | #CloudData                      |
| Changed          | #BigQuery        | #DataPlatform                   |
| Fix / Issue      | #BigQuery        | #GCP                            |
| Deprecated       | #BigQuery        | #MigrationGuide                 |
| ML / AI Feature  | #GenerativeAI    | #VertexAI                       |
| Analytics        | #DataAnalytics   | #BI                             |
| Streaming        | #DataStreaming   | #Dataflow                       |
| Security / IAM   | #CloudSecurity   | #GCP                            |
| Pricing / Quota  | #GoogleCloud     | #CostOptimization               |

## Rules
- Max 3 hashtags total per tweet (including #BigQuery)
- Never use generic tags like #Tech, #AI without a specific BigQuery context
- Prefer community-used tags over invented ones
- For GA announcements, add `#NowGA` as a bonus tag if character limit allows
